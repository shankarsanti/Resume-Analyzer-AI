## Workspace Configuration

This is a **monorepo** using npm workspaces with three packages:

1. **frontend** - React application
2. **backend** - Express API server
3. **shared** - Shared TypeScript types

## Key Files

### Root Level

- **package.json**: Defines workspaces and root-level scripts
- **vercel.json**: Vercel deployment configuration for frontend
- **.env**: Environment variables (not committed)
- **.gitignore**: Files and folders to ignore in git

### Frontend

- **src/**: All frontend source code
- **dist/**: Production build output (generated by Vite)
- **vite.config.ts**: Vite bundler configuration
- **vitest.config.ts**: Test configuration
- **tailwind.config.js**: Tailwind CSS configuration

### Backend

- **src/**: All backend source code and tests
- **dist/**: Compiled TypeScript output (generated by tsc)
- **tsconfig.json**: TypeScript compiler configuration
- **vitest.config.ts**: Test configuration

### Shared

- **src/index.ts**: Shared TypeScript types and interfaces
- **dist/**: Compiled output used by frontend and backend

## Build Artifacts (Gitignored)

The following directories are generated during build and are not committed:

- `node_modules/` - Dependencies
- `dist/` - Build outputs
- `coverage/` - Test coverage reports
- `.env` - Environment variables

## Scripts

Run from the root directory:

```bash
# Development
npm run dev:frontend    # Start frontend dev server
npm run dev:backend     # Start backend dev server

# Building
npm run build:shared    # Build shared package
npm run build:frontend  # Build frontend
npm run build:backend   # Build backend
npm run build           # Build all packages

# Testing
npm test                # Run all tests

# Code Quality
npm run lint            # Lint all packages
npm run format          # Format all code
```

## Environment Variables

### Frontend (.env in frontend/)
- `VITE_API_URL` - Backend API URL

### Backend (.env in backend/)
- `PORT` - Server port (default: 5000)
- `NODE_ENV` - Environment (development/production)
- `FRONTEND_URL` - Frontend URL for CORS

## Deployment

- **Frontend**: Deployed to Vercel (configured via vercel.json)
- **Backend**: Can be deployed to Railway, Render, or similar Node.js hosting
