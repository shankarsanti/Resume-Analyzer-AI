STEP 1: Deploy Frontend (2 minutes)
────────────────────────────────────────────────────────────────────────────────

Go to Vercel Dashboard: https://vercel.com/dashboard

1. Select your project
2. Go to "Deployments" tab
3. Cancel any running builds
4. Click "..." on any deployment → "Redeploy"
5. In the modal:
   ✅ CHECK "Use latest commit from Git"
   ❌ UNCHECK "Use existing Build Cache"
6. Click "Redeploy"

Expected Result: Build succeeds, deployment shows "Ready" ✅

────────────────────────────────────────────────────────────────────────────────
STEP 2: Set Environment Variables (1 minute)
────────────────────────────────────────────────────────────────────────────────

Go to: Settings → Environment Variables

Add these:
  VITE_API_URL = https://your-backend-url.com
  VITE_NODE_ENV = production

(Use temporary URL if backend not ready yet)

────────────────────────────────────────────────────────────────────────────────
STEP 3: Deploy Backend (5 minutes)
────────────────────────────────────────────────────────────────────────────────

Option A - Railway (Recommended):

  npm i -g @railway/cli
  railway login
  cd packages/backend
  railway init
  railway up
  railway domain  # Copy this URL

Option B - Render:

  1. Go to https://render.com
  2. New → Web Service
  3. Connect GitHub repo
  4. Root Directory: packages/backend
  5. Build Command: npm install && npm run build
  6. Start Command: npm start

────────────────────────────────────────────────────────────────────────────────
STEP 4: Update Frontend with Backend URL (1 minute)
────────────────────────────────────────────────────────────────────────────────

1. Go to Vercel → Settings → Environment Variables
2. Update VITE_API_URL with your actual backend URL
3. Redeploy frontend
